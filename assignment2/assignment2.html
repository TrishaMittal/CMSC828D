Dataset

The Chicago Crimes Dataset contains the details of the reported incidents from 2001 onwards in Chicago. Data is extracted from the Chicago Police Department's CLEAR (Citizen Law Enforcement Analysis and Reporting) system. There are 6,709,800 records in the dataset. We wanted to explore and discover the underlying patterns in the data, uncover the relations among different attributes of the dataset and also relate them to real-world scenarios. This would be a good practice of visualisation for us, as we are new to this data, we will go in an iterative loop over making insights and providing better visualisations.

The data is available online at: Dataset url.

Evaluation of Tool

Strengths

STRENGTH 1: The process of generating visualisations was very simple as Tableau is UI-centric and not script-centric. The process is fast with good response times. Switching between different types of encodings is very quick and hence enables one to settle on the one which could give the most insight into the data.

STRENGTH 2: Splitting of attributes is extremely easy. Moreover, Tableau can understand attribute-types like Date. For instance, we had a attribute named Date, which contained date in the format mm/dd/yyyy along with the timestamp in the format hh:mm:ss AM/PM. While doing the visualisations, with a simple drop-down menu, we were able to choose if we wanted to plot the date, in year, month, date or day format.

STRENGTH 3: Combining two views with the help of dashboard made the process of exploration and deriving conclusions even easier. For instance, it was easier to make comparisons about crime trends in different districts. We have added screenshots for this in the previous section, wherever we exploited this feature of Tableau.

Weaknesses

WEAKNESS 1: Whatever plethora of features that Tableau provides us with are extremely good and make the visualisation process fun. But, if one wants to try and experiment something out of this, one can face limitations. For instance, to try and make line graph, it is mandatory that one of the fields has to be Date. That's in a way a limitation.

WEAKNESS 2: While experimenting with a different dataset, we realised that though Tableau supports datasets in .json format, joining two tables directly in .json is not permitted. Though there is a work around it using blending. This shows that not all features are present for all type of data sources

WEAKNESS 3: Tableau Desktop is currently available only for Mac and Windows, and not for Linux. Moreover, not being an open-source software the documentation and the Tableau user community is limited.
